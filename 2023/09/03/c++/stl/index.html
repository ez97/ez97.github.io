刷算法前复习一下常用STL的使用方法

做题还是效率优先，奇技淫巧等我medium随便过了再考虑……

不能再沉迷造轮子了

Iterators

C++的一个设计思路是，将container和algorithm分开设计，然后用iterator充当算法操纵容器的中介

从功能表现上，迭代器类似于指针，拥有指针的引用（reference）和解引用（deference）能力

相较于原生指针（native pointer），包装后迭代器更加安全，解决了内存漏洞问题（写循环的时候不会再因为看错一个数字就触发了segmentation fault了）

迭代器可分为四种：

所有迭代器都有的功能有:

  • 类似于指针解引用，*iterator表示迭代器指向的元素（const类型无法修改）
  • iterator++代表指向“下一个”元素，区别在于正向是后一个，反向（reverse）是前一个

不过在学习数据结构时我们知道，有一些数据结构，是不支持随机访问的，对其中的数据存取会受到限制（如list），这一点也体现在了迭代器上，根据容许程度可能还有以下功能：

  • 对于双向迭代器：p--也是有定义的
  • 对于随机访问迭代器：p+=i，p+i，p[i]，p1<p2，p2-p1都是有意义的

在常用的容器里，

拥有随机访问迭代器的容器有：vector、deque

拥有双向迭代器的容器有：list、set、map

不支持迭代器的容器有：stack、queue、priority_queue

Containers

常用的容器操作有：

1.vector

std::vector<type> name

实现方式为动态数组，在内存中连续存储，提供快速随机访问能力

当向量元素大于容器容量时自动扩充，可用c.capacity()进行查询

2.queue

std::queue<type> name

queue的实现依赖其底层容器的选择，默认为deque，也可以选择list

deque为FIFO队列，存储方式为动态分配的内存块，优势在于在序列头部/尾部快速添加、删除元素(O(1))，缺点在于队列中删除/插入元素需要移动整块存储空间

list为双向链表，存储方式为分布节点，优势在于能在已知位置快速插入、删除元素(O(1))，缺点在于随机访问需要顺序遍历

3.priority_queue

std::priority_queue<type> name

优先队列，在queue的基础上增加了一个比较函数，使最大元素位于前列（First in， Largest out）

实现方式为heap

4.pair

std::pair <type1, type2> name

键值对，通过p.first和p.second来访问两个元素

实现方式为class

5.map

std::map<key_type, value_type> name

有序关联容器，元素默认按value升序排序，可以通过map1.insert(pair1<type1, type2>)来将pair插入

实现方式为红黑树，查找、插入、删除的时间复杂度均为O(nlogn)

6.set

std::set<type> name

有序唯一关联容器，元素默认升序排序，容器中所有元素值唯一

实现方式为红黑树

7.stack

std::stack<type> values

栈，后进先出数据结构，默认基础容器为deque

一般常用s.push(val)和s.pop()操作
